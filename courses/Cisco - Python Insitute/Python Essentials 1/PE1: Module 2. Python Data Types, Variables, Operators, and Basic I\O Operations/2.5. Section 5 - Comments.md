## What is a comment?

A remark in the code ignored by Python at runtime — written for humans, not the interpreter.

```python
# This is a comment — Python ignores this line
x = 10  # inline comment — also ignored
```

---

## Rules

- Starts with `#` — extends to the end of the line
- For multi-line comments, use `#` on each line
- No block comment syntax in Python (no `/* */`)

```python
# This program calculates the hypotenuse.
# a and b are the lengths of the legs.
a = 3.0
b = 4.0
c = (a ** 2 + b ** 2) ** 0.5  # ** instead of square root
print("c =", c)
```

---

## Uses

**1. Explain non-obvious logic**
```python
x //= 2  # floor division keeps result as int
```

**2. Temporarily disable code (debugging)**
```python
x = 1
y = 2
# y = y + x   ← disabled for testing
print(x + y)  # 3
```

**3. Document authorship or context**
```python
# Author: Ruben Jiménez
# Date: 2025-01-15
# Purpose: ETL pipeline for sales data
```

> 💡 Shortcut: `Ctrl + /` (Windows) or `Cmd + /` (Mac) to toggle comments on selected lines.

---

## Self-commenting names

The best comment is often a good variable name — no comment needed.

```python
# ❌ unclear
a = 3.0
b = 4.0

# ✅ self-commenting
leg_a = 3.0
leg_b = 4.0
hypotenuse = (leg_a ** 2 + leg_b ** 2) ** 0.5
```

---

## Common mistakes

```python
# ❌ Multi-line comment without # on each line
# This is
a multiline   # ← SyntaxError!
comment. #

# ✅ Correct multi-line comment
# This is
# a multiline
# comment.

# ❌ Wrong or misleading comment
x = x + 1  # subtract 1 from x  ← lying comment, dangerous!
```

## Quiz Answers

<details>
<summary>Q1 — # print("String #1") / print("String #2")</summary>

```
String #2
```
The first line is commented out — only the second `print()` runs.
</details>

<details>
<summary>Q2 — # This is / a multiline / comment. # / print("Hello!")</summary>

```
SyntaxError: invalid syntax
```
Only the first line is a valid comment. `a multiline` is interpreted as code — `a` is a variable but `multiline` after it is invalid syntax.
</details>
